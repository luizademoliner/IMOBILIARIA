# 🏠 Projeto Integrador - Sistema para Imobiliária

Este repositório contém o projeto desenvolvido por **Luiza Demoliner** e **Bianca Tremea**, como avaliação da disciplina de **Linguagem de Programação I (LP I)** no curso técnico de **Desenvolvimento de Sistemas - IFSul**, realizado em 2024.

## 📌 Sobre o projeto

O sistema foi desenvolvido com o objetivo de aplicar os conhecimentos adquiridos ao longo do semestre, principalmente sobre **Java (Programação Orientada a Objetos - POO)** e **Banco de Dados**.

Criamos uma solução para auxiliar a organização do trabalho de uma **imobiliária**, focando na gestão de **corretores**, **clientes** e **imóveis**.

### 🧑‍💼 Funcionalidades principais:

- Cadastro de **corretores** com seus dados de contato
- Cadastro de **clientes**, incluindo aqueles que desejam alugar, comprar ou anunciar imóveis
- Registro de **imóveis** disponíveis para venda ou locação, com detalhes como localização, características e preço
- Associação de cada imóvel a um **corretor responsável**
- Controle de **contratos de venda ou locação**
- Anúncio de imóveis por parte dos clientes, por meio da contratação de serviços da corretora
- **Permissões diferenciadas** para clientes e corretores, garantindo a segurança e organização das informações

## ✅ Requisitos atendidos

1. Permitir ao usuário **Cliente** realizar seu próprio cadastro
2. Permitir ao **Cliente** contratar os serviços da corretora para anunciar um imóvel
3. Permitir ao **Cliente** realizar locação ou compra de imóveis
4. Permitir ao **Corretor** acessar a lista de corretores cadastrados
5. Permitir ao **Corretor** acessar a lista de clientes cadastrados
6. Permitir ao **Corretor** excluir imóveis e clientes cadastrados
7. Permitir ao **Corretor** editar anúncios de imóveis
8. Permitir que **Clientes** e **Corretores** acessem a lista de imóveis disponíveis

## 🛠️ Tecnologias utilizadas

- **Java (POO)**
- **MySQL** (Banco de Dados)
- **JDBC** (conexão Java com o banco)
- **Visual Studio Code (VS Code)** como IDE

## 🚀 Possíveis melhorias futuras

Temos interesse em continuar evoluindo este projeto, com as seguintes ideias de melhoria:

- Desenvolvimento de uma interface **front-end**, tornando o sistema mais amigável e visualmente agradável para os usuários
- Melhorias no **back-end**, com otimização de código, melhor organização das classes e inclusão de novas funcionalidades
- Implementação de uma camada de **validação de dados** mais robusta
- Adição de recursos para melhorar a experiência do usuário, como mensagens de confirmação e tratamento de erros mais detalhado

## 🎯 Objetivo pedagógico

Este projeto teve como principal objetivo reforçar o aprendizado de:

- Manipulação de dados em **banco de dados relacional**
- Aplicação dos conceitos de **Programação Orientada a Objetos (POO)** em Java
- Implementação de um sistema com diferentes **níveis de usuário** (clientes e corretores)
- Conexão entre **aplicação Java** e **banco de dados**
